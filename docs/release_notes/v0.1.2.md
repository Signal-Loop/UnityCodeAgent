# Unity Code Agent 0.1.2 Release Changes

## TL;DR

- Choose GitHub Copilot or a bring-your-own-key (BYOK) provider explicitly in Unity Code Agent settings.
- Provider settings are now isolated, reducing the chance that stale custom-provider values affect Copilot sessions.
- BYOK setup gives clearer validation and guidance before a chat session starts.

## ✨ What's New for You?

The 0.1.2 release makes provider setup clearer and safer, so you can switch between GitHub Copilot and an OpenAI-compatible endpoint with confidence.

- **Explicit provider selection:** Settings now include a provider selector for GitHub Copilot and BYOK. The inspector shows the BYOK URL and API-key fields only when BYOK is selected.
- **Cleaner Copilot sessions:** When Copilot is selected, saved BYOK URL and API-key values are excluded from the session configuration.
- **Stronger BYOK validation:** A BYOK provider now requires a complete HTTPS base URL before Unity Code Agent creates its configuration.
- **Improved model guidance:** If no model is selected, Unity Code Agent now directs you to select one and retry.

---

## ⚠️ Breaking Changes & Migration

Most users can upgrade directly. To use an OpenAI-compatible provider, open Unity Code Agent settings, select **BYOK**, then enter a complete HTTPS base URL and your API key.

---

## 🚀 Key Improvements

### ⚙️ Provider configuration

- **Separated provider modes:** Copilot and BYOK now have distinct configuration paths, preventing a prior BYOK configuration from being sent with a Copilot session.
- **Focused settings UI:** BYOK-specific fields appear only when they apply, making the normal Copilot setup less cluttered.
- **Actionable validation:** Invalid or missing BYOK URLs are reported before provider configuration is created.

### 🛠️ Developer workflow

- **Solution-file support:** Added `UnityCodeAgent.slnx` for opening the Unity Code Agent editor projects in standard .NET tooling.
