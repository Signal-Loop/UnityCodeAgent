# Filter sessions by project session id
- status: Completed
- order: 1000
- goal: Show and operate on sessions for the current Unity project only by parsing project identity from Unity-created session ids, verified by focused Unity-side behavior tests, while preserving create/open/send/abort access to sessions by explicit id.

- steps:
    - [x] Add a shared helper for Unity-created session id creation and parsing
    - [x] Add Unity-side session-list filtering by parsed session id project identity

Original note:
Filter should work on unity side and be based on session id project root path.

User review note:
- Do not base filtering on `Cwd` or `GitRoot`; use session id parts. Create a shared helper to create/read session ids if one does not already exist.

Research:
- There is no shared session id helper today. Session ids are currently created by private `AgentService.CreateSessionId(UnityCodeAgentPaths paths, DateTimeOffset timestampUtc)`.
- Current Unity-created ids use the shape `UnityCodeAgentSession-{yyyyMMddHHmmssfff}-{SafeProjectRoot}` when `SafeProjectRoot` is available, where `SafeProjectRoot` is generated by replacing every non-alphanumeric project-root character with `_`.
- `UnityCodeAgentPaths.SafeProjectRoot` is already the project identity component available on the Unity side.
- `SessionSummaryDto.Cwd` and `GitRoot` exist, but they should not be used for this task's filtering rule.
- `AgentService.GetSessionsAsync` currently returns the service list unchanged. This is the narrowest Unity-side place to filter before both current-session selection and the sessions view consume the list.
- `ChatEditorWindowClient.ShowSessionsAsync` passes through `AgentService.GetSessionsAsync`; keeping filtering in `AgentService` avoids duplicating behavior in UI code.
- `SessionRequestFactory` uses `context.Paths.ProjectRoot` as the working directory for create/open requests. Existing create/open/send/abort flows should continue to address sessions by `SessionId` without applying an extra id guard.

Plan:
- Add a shared Unity-side helper, likely under `Packages/com.signal-loop.unitycodeagent/Editor/Service` or `Editor/Infrastructure`, for Unity session id handling.
- Move session id creation out of private `AgentService.CreateSessionId(...)` into the helper without changing the generated id format.
- Add a parser such as `TryParse(string sessionId, out UnityCodeAgentSessionId parts)` that recognizes the Unity-created prefix, timestamp part, and project identity part. It should reject malformed ids rather than guessing.
- Use `context.Paths.SafeProjectRoot` as the expected project identity, and keep only sessions whose parsed project identity equals it.
- Do not use `SessionSummaryDto.Cwd` or `GitRoot` in the filter.
- Decide explicitly how to handle legacy or external sessions that do not parse. Default to excluding them from the project-scoped session list unless a compatibility case is required during implementation.
- Apply the filter only in `GetSessionsAsync`. Do not block `OpenSessionAsync`, `SendPromptAsync`, or `AbortPromptAsync`; those operations must keep working by explicit session id.
- Log the pre-filter and post-filter counts at debug level so unexpected missing sessions are diagnosable without changing the UI.

Verification:
- Add focused EditMode tests for the shared helper: creates the existing id format, parses valid ids, extracts the project identity, and rejects malformed/non-Unity ids.
- Add focused EditMode coverage around `AgentService.GetSessionsAsync` with fake API data containing current-project ids, other-project ids, malformed ids, and non-Unity ids.
- Add or update a `ChatEditorWindowClient.ShowSessionsAsync` behavior check only if `AgentService` tests do not already cover the user-visible path.
- Run targeted Unity EditMode tests for the touched service/client test fixture.

Completion:
- Added `UnityCodeAgentSessionIds` to create the existing Unity session id format and parse timestamp/project identity from Unity-created ids.
- `AgentService.GetSessionsAsync` now filters returned sessions to ids whose parsed project identity matches `context.Paths.SafeProjectRoot`, excluding malformed, non-Unity, and other-project ids without using `Cwd` or `GitRoot`.
- Current-session selection applies the same project filter before opening the first session, while explicit `OpenSessionAsync`, `SendPromptAsync`, and `AbortPromptAsync` remain id-based and unguarded.
- Verification passed: focused Unity EditMode run with 11 passing tests covering the new helper, session filtering, explicit id operations, and existing session id creation behavior.
